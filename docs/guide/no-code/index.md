# Start here if you don't write code

This is the gentlest way into KB Factory. It's written for people who **don't
program** and don't want to touch a terminal — you'll do everything by *talking
to your assistant* in plain language, the same way you already chat with it.

If you're comfortable with a command line, you may prefer the
[main User Guide](../index.md), which is a little more technical. Everything here
also lives there — this track just takes it slower and skips the jargon.

## What KB Factory does, in one sentence

It gives your project a **memory** your AI assistant can use: the decisions you
made, the facts you learned, the things still open — kept safely, so the next
conversation picks up where the last one left off instead of starting from
scratch.

## The whole idea in 20 seconds

- You **talk** to your assistant. ("Remember that we decided to use weekly
  releases." / "What did we decide about pricing?")
- It **writes things down** for you in the project's memory, and **reads them
  back** when you ask.
- When something changes, the old version is **kept**, not erased — so you can
  always see *what changed and why*.

That's it. Ready? Start with [what this is](what-this-is.md), or jump straight
into [your first session](first-session.md).
